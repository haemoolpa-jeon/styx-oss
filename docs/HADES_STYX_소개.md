Styx 관심이 많으셔서 간단한 설명 추가

생각 했던 것 보다 관심을 많이 받아서 부담 스럽네여 ㅠㅠ
간단하게 구상-구현 중인것만 설명 좀 드리겠습니다.

왜 직접 만들고자 하는가...?

SyncRoom (야마하)
- 일본 서버 위주라 한국에서 지연이 큼
- 무료 버전은 기능 제한
- 불특정 다수를 위한 서비스라 트래픽 경쟁
- 커스터마이징 불가능

SonoBus
- 설정이 복잡함
- 서버를 직접 운영해야 함
- UI가 직관적이지 않음
- 안정성 이슈가 종종 발생

Styx가 위 두개의 프로그램 대비 추구 하고자 하는것

1. 🎯 소규모 전용 = 쾌적한 환경

사용자 수 - 8명 (HADES 전용)
서버 부하 - 거의 없음
트래픽 경쟁 - 없음
대역폭 - 하데스만 씀

2. ⚡ 지연 시간 비교 (공개된 정보에 따라)

SyncRoom - 50~100ms 
Styx - 20~40ms 를 노력 하고 있습니다만... 아마 더 100~150ms 정도가 여유롭지 않을까... 싶네요 ㅠ

> 💡 참고: 사람이 "동시에 악기를 연주한다"고 느끼려면 지연이 30ms 이하여야 합니다.
> 50ms까지는 약간의 적응으로 합주 가능, 100ms 이상은 어렵다고 합니다.
> 악기가 아닌 음성은 비교적으로 조금더 여유가 있다고 하네요

3. 🎛️ 개인별 설정
- 에코 제거 ON/OFF
- 노이즈 제거 ON/OFF  
- 자동 게인 컨트롤
- PTT (Push-to-Talk) 모드
- 개인별 볼륨 조절

🖥️ 데스크톱 앱 (개발 중)

데스크톱 앱:
마이크 → [ASIO 직접 접근] → UDP → 인터넷 → UDP → [ASIO 직접 접근] → 스피커
              ↑ 최소 지연                           ↑ 최소 지연


- ASIO 드라이버 지원 (오디오 인터페이스 사용자용)
- Custom UDP 모드 (WebRTC보다 빠름)
- Opus 코덱 (고음질 + 저대역폭)

자주 묻는 질문

Q: 인터넷 속도가 느려도 되나요?
> 10Mbps 이상이면 충분합니다. 오디오는 생각보다 데이터가 적어요.

Q: 오디오 인터페이스가 있으면 더 좋나요?
> 데스크톱 앱에서 ASIO 드라이버를 사용하면 지연이 더 줄어듭니다.
> 없어도 사용은 가능합니다.
